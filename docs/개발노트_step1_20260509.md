## 프로젝트 개요

FastAPI 기반 웹 모니터링 시스템. DART, 금융위원회 데이터를 수집하여 키워드 매칭과 이메일 알람 제공.

**Step 1 목표:**
- FastAPI 기초 구조
- RSS 수집 파이프라인 (DART, FSC)
- 파일 기반 데이터 저장 (JSONL)
- 키워드 매칭 엔진
- 이메일 + 파일 알람

|---|------|------|

| 2 | `app/models.py` | Pydantic 모델 — `ExternalEvent`, `AlertLog`, `KeywordRule`, `SystemState` (`to_jsonl/from_jsonl`) |
| 3 | `app/database.py` | JSONL I/O — `append_jsonl`, `load_jsonl(limit)`, `save/load_state`, `compute_content_hash`, `event_exists`, `load_existing_hashes` (per-file `threading.Lock`) |
| 4 | `config/sources.yaml` | DART + FSC RSS 소스 정의 |

| 7 | `monitor/collectors/__init__.py` | 패키지 마커 |
| 8 | `monitor/collectors/rss.py` | `RSSCollector` — `fetch()` (feedparser + UA), `parse()` (RSS/Atom), `collect()` (try/except) |
| 9 | `monitor/collectors/dart.py` | `DARTCollector` — `DART_WATCHLIST` env 기반 corp_code 필터링 |

| 13 | `monitor/worker.py` | `Worker` — 수집 → 중복 체크 → 매칭 → 발송 → JSONL append, in-memory dedup cache |

| 15 | `app/routes/__init__.py` | 패키지 마커 |
| 16 | `app/routes/status.py` | `GET /status`, `GET /events?limit=N`, `GET /alerts?limit=N`, `POST /trigger` |
| 17 | `main.py` | FastAPI 앱 — lifespan으로 worker + scheduler 시작/종료 |
