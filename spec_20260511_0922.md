# spec_20260511_0922.md — 사이트별 체크박스 구독 (Watchtower Step 4.5)

> **레벨**: Step 4 deployed 이후 **incremental cycle** (마이너). Step 5 진입 전 ad-hoc 보강.
> **타깃**: 사이드바에 사이트별 체크박스 + AND 알림 필터.
> **참조 ideation**: `ideation/ideation_사이트별구독_20260511_0922.md`
> **EARS 패턴**: Ubiquitous (SHALL), Event-driven (WHEN), Unwanted (IF…THEN), State-driven (WHILE), Optional (WHERE)

---

## 1. Overview

Watchtower 사이드바에서 사용자가 카테고리 단위 ⭐/🔔 구독에 더해, **카테고리 내 사이트마다 체크박스로 개별 모니터링 ON/OFF**를 결정할 수 있게 한다. 알림은 카테고리 구독과 사이트 구독의 **AND** 조건으로 발송된다.

- 모델: `Subscription` 테이블을 `category_subscriptions`로 리네이밍 + 신규 `site_subscriptions` 테이블 도입.
- API: 경로 2개 리네이밍, 신규 3개 (site GET/PATCH + 옵션 조인 GET).
- 알림: instant·digest 양쪽 SQL에 `JOIN site_subscriptions WHERE enabled=True` 추가.
- UI: 사이드바 accordion + 사이트 li + 체크박스 + 상태 배지.
- 마이그: 부팅 1회성 함수가 옛 `subscriptions` 테이블을 `category_subscriptions`로 이전 후 DROP.

---

## 2. Glossary

| 용어 | 정의 |
|---|---|
| **카테고리 구독(CategorySubscription)** | 사용자 × 카테고리 구독 row. `subscribed`(별), `channel`(벨: instant/digest/off). 기존 `Subscription` 테이블 리네이밍. |
| **사이트 구독(SiteSubscription)** | 사용자 × 사이트 모니터링 ON/OFF row. `enabled: bool`. **신규**. |
| **system_enabled** | `Site.enabled` 컬럼. 관리자가 비활성화한 사이트. 사용자 PATCH 우회 불가. |
| **AND 조건** | 알림 발송 조건 = `CategorySubscription.subscribed=TRUE AND channel∈{instant,digest} AND SiteSubscription.enabled=TRUE AND Site.enabled=TRUE`. |

## 3. Functional Requirements (EARS)

### 3.1 FR-SUB-006 — 사이트 구독 토글 (Event-driven)

**WHEN** 사용자가 사이드바에서 사이트 체크박스를 토글
**THE system SHALL** `PATCH /api/site-subscriptions/{site_id}` 호출을 수신하여 `(user_id, site_id)` row를 **upsert**하고, `enabled` 필드를 요청값으로 갱신한다.

**입력 스키마**:
```json
{ "enabled": true|false }
```

**출력 (200 OK)**:
```json
{
  "site_id": "s17",
  "enabled": true,
  "updated_at": "2026-05-11T09:22:00+00:00"
}
```

**제약**:
- 동일 `(user_id, site_id)` 중복 row 금지 (`UNIQUE(user_id, site_id)`).
- `updated_at`은 서버 측 자동 갱신.

---

### 3.2 FR-SUB-007 — system disabled 사이트 보호 (Unwanted)

**IF** 사용자가 `Site.enabled=FALSE` 사이트에 대해 `PATCH /api/site-subscriptions/{site_id}` 를 `{enabled: true}` 로 호출
**THEN THE system SHALL** HTTP **422 Unprocessable Entity** 를 반환하고, `site_subscriptions` row를 변경하지 **않는다**.

**에러 응답**:
```json
{
  "detail": "site_id 's21' 은 관리자에 의해 비활성화되어 사용자가 활성화할 수 없습니다."
}
```

**참고**: `{enabled: false}` 호출은 system_enabled 상태와 무관하게 항상 허용 (사용자가 끄는 것은 안전).

---

### 3.3 FR-SUB-008 — 사이트 구독 조회 (Ubiquitous)

**THE system SHALL** `GET /api/site-subscriptions` 호출에 대해 `_resolve_me()` 사용자의 사이트 구독 상태를 반환한다.

**출력 (200 OK)**: 현재 시드된 `Site.id` 30개 전부에 대해 1행씩(누락 시 default `enabled=False` fill):
```json
[
  { "site_id": "s1", "enabled": false, "updated_at": null },
  { "site_id": "s2", "enabled": true,  "updated_at": "2026-05-11T..." },
  ...
]
```

---

### 3.4 FR-SUB-009 — 카테고리별 사이트 조인 응답 (Optional)

**WHERE** UI가 첫 로드 시 사이드바 렌더링을 단순화하려는 경우,
**THE system SHALL** `GET /api/categories/{cid}/sites` 호출에 대해 해당 카테고리 소속 사이트와 사용자 구독 상태를 조인한 응답을 반환한다.

**출력 (200 OK)**:
```json
{
  "category_id": "ai",
  "sites": [
    {
      "id": "s17",
      "name": "Anthropic News",
      "url": "https://www.anthropic.com/news",
      "status": "ok",
      "system_enabled": true,
      "enabled_user": true
    },
    ...
  ]
}
```

**에러**: 미존재 `cid` → HTTP **404**.

### 3.5 FR-NOTIF-009 — AND 조건 알림 필터 (Ubiquitous)

**THE system SHALL** 모든 알림(`send_instant`, `send_digest`)을 다음 4개 조건이 **모두 TRUE**인 경우에만 발송한다:

1. `Site.enabled = TRUE` (system_enabled)
2. `CategorySubscription.subscribed = TRUE` (사용자 카테고리 별 ON)
3. `CategorySubscription.channel ∈ {'instant', 'digest'}` (채널 매칭)
4. `SiteSubscription.enabled = TRUE` (사용자 사이트 ON)

**적용 위치**:
- `monitor/watchtower/notifier.py:send_instant` — Item 단위 발송 시 위 4 조건 SQL JOIN.
- `monitor/watchtower/notifier.py:send_digest` — 09:00 KST cron 묶음 시 동일.

**SQL 예시** (instant):
```sql
SELECT U.* FROM users U
  JOIN category_subscriptions CS ON CS.user_id = U.id
  JOIN site_subscriptions     SS ON SS.user_id = U.id
  JOIN sites                  S  ON S.id = SS.site_id
WHERE CS.category_id = :cat_id
  AND CS.subscribed  = TRUE
  AND CS.channel     = 'instant'
  AND SS.site_id     = :site_id
  AND SS.enabled     = TRUE
  AND S.enabled      = TRUE
```

**AlertLog**: 위 조건 미충족으로 발송이 누락된 경우 AlertLog row 생성 **안 함** (skipped 케이스가 아니라 "구독자 없음"으로 해석).

### 3.6 FR-MIG-001 — 부팅 1회 마이그레이션 (Event-driven)

**WHEN** 애플리케이션 부팅 시(`lifespan` 진입), DB에 옛 `subscriptions` 테이블이 존재하고 신규 `category_subscriptions` 테이블이 비어 있는 경우,
**THE system SHALL** 옛 테이블의 모든 row를 `category_subscriptions`로 SELECT→INSERT 복사한 뒤 옛 테이블을 DROP한다. 작업은 단일 트랜잭션 안에서 수행한다.

**멱등성**: 옛 테이블이 이미 없으면 NO-OP. 신규 테이블에 row가 이미 있으면 NO-OP (기 마이그된 상태로 간주).

**롤백**: 마이그 중 예외 발생 시 트랜잭션 롤백, 옛 테이블 보존, 부팅 실패 처리하지 말고 WARN 로그 후 그대로 진행 (사용자가 수동 복구 가능).

**로그**:
- 성공: `INFO [migration] subscriptions → category_subscriptions: N rows migrated`
- NO-OP: `INFO [migration] subscriptions table not found — skip`
- 실패: `WARN [migration] failed: {exc_type}; continuing with current schema`

---

### 3.7 FR-MIG-002 — 시드 멱등 보강 (Ubiquitous)

**THE system SHALL** 부팅 시 시드 단계에서 다음 두 함수를 멱등 실행한다:
- `_seed_category_subscriptions(session)` — 기존 `_seed_subscriptions` 리네이밍. 사용자 × 카테고리 카르테시안, default `subscribed=False, channel='off'`. **1×8=8 row**.
- `_seed_site_subscriptions(session)` — 신규. 사용자 × 사이트 카르테시안, default `enabled=False`. **1×30=30 row**.

**멱등**: 이미 존재하는 `(user_id, category_id)` 또는 `(user_id, site_id)` 쌍은 SKIP.

---

### 3.8 FR-UI-006 — 사이드바 accordion + 체크박스 (Event-driven)

**WHEN** 사용자가 카테고리 행의 `▶` 토글을 클릭
**THE system SHALL** 해당 카테고리의 사이트 목록을 펼치고 (`▼`로 변환), 다시 클릭 시 접는다.

**WHEN** 사용자가 사이트 체크박스를 클릭
**THE system SHALL** 낙관적 UI 업데이트 후 `PATCH /api/site-subscriptions/{site_id}` 비동기 호출하고, 실패 시 이전 상태로 rollback + 한국어 토스트 표시.

**초기 상태**:
- "내 구독 카테고리" 영역(`subscribed=True` 카테고리): 모두 expand.
- "기타 카테고리" 영역(`subscribed=False`): 모두 collapse.
- accordion 펼침 상태 영속 **하지 않는다** (localStorage 미사용 정책 유지).

---

### 3.9 FR-UI-007 — system disabled 사이트 표시 (Unwanted)

**IF** `Site.enabled=FALSE`,
**THEN THE system SHALL** UI에서 해당 사이트 체크박스를 `disabled` + 회색 표시하고, hover 시 tooltip `"관리자에 의해 비활성화됨"`을 보여준다.

**클릭 동작**: 체크박스 클릭 무반응 (브라우저 기본 disabled 동작). 사용자가 어떤 방법으로든 PATCH 호출에 성공해도 서버 측 FR-SUB-007이 422로 거부.

---

### 3.10 FR-UI-008 — 사이트 상태 배지 (Ubiquitous)

**THE system SHALL** 각 사이트 행에 `Site.status` 값을 작은 색 점(●)으로 표시한다.

| 값 | 색 | 의미 |
|---|---|---|
| `ok` | 녹색 | 마지막 크롤 성공 |
| `delayed` | 노란색 | 연속 1~2회 실패 |
| `failed` | 빨간색 | 연속 3~4회 실패 |
| `blocked` | 회색 | 5회 연속 실패 또는 robots.txt 차단 |

배지는 정보 표시만 — 클릭/상호작용 없음.

## 4. Data Model

### 4.1 신규 테이블 — `site_subscriptions`

```python
class SiteSubscription(Base):
    __tablename__ = "site_subscriptions"
    __table_args__ = (
        UniqueConstraint("user_id", "site_id", name="uq_user_site"),
    )

    id: Mapped[str] = mapped_column(String(32), primary_key=True, default=lambda: uuid.uuid4().hex)
    user_id: Mapped[str] = mapped_column(ForeignKey("users.id"))
    site_id: Mapped[str] = mapped_column(ForeignKey("sites.id"))
    enabled: Mapped[bool] = mapped_column(Boolean, default=False)
    updated_at: Mapped[datetime] = mapped_column(
        DateTime(timezone=True),
        default=lambda: datetime.now(timezone.utc),
        onupdate=lambda: datetime.now(timezone.utc),
    )
```

### 4.2 리네이밍 — `subscriptions` → `category_subscriptions`

- 클래스명: `Subscription` → `CategorySubscription`
- 테이블명: `subscriptions` → `category_subscriptions`
- 컬럼/제약 변경 없음. UNIQUE(user_id, category_id) 유지.

## 7. Out of Scope (이번 사이클 제외)

> **2026-05-11 보강 (incremental cycle)**: 아래 5개 중 2개를 즉시 처리, 1개는 헬퍼만 선제 구현, 2개는 명시적 부결로 확정.

- ~~**사이트 등록 API 트리거**~~ → **헬퍼 선제 구현** (FR-MIG-003): API 본체는 Step 5지만 호출될 헬퍼 함수 `create_default_site_subscriptions_for_new_site(session, site_id)`를 `app/db/seed.py`에 추가. 사이트 등록 API가 lazy하게 활용.
- ❌ **카테고리 별 토글 시 사이트 일괄 ON/OFF 매크로** — 사용자 결정 7-a 명시 부결, 영구 보류.
- ~~**Accordion 펼침 상태 영속**~~ → **sessionStorage 영속** (탭 단위, FR-UI-009): localStorage 정책 위반 안 함. 새 탭/새 창에서는 default rule(구독 expand, 기타 collapse) 적용.
- ❌ **사이트별 채널 분기** — 사용자 결정 부결.
- **다중 사용자 SiteSubscription 확장** — Phase 2 인증/SSO 사이클 유지.

### 7-1. FR-MIG-003 — 사이트 등록 시 default SiteSubscription 자동 생성 (헬퍼)

**WHEN** 사이트 등록 API(Step 5 FR-SITE-002)가 새 `Site` row를 commit한 후,
**THE system SHALL** `create_default_site_subscriptions_for_new_site(session, site_id)`를 호출하여 모든 사용자에 대해 1행씩 `SiteSubscription`을 멱등 생성하며, 각 행의 `enabled`는 해당 사용자의 `CategorySubscription.subscribed` (그 사이트의 카테고리)와 동일하게 설정한다.

**멱등성**: 이미 (user_id, site_id) row가 존재하면 SKIP. 사이트 row 미존재면 NO-OP.

### 7-2. FR-UI-009 — Accordion 펼침 sessionStorage 영속

**WHILE** 동일 탭이 열려 있는 동안,
**THE system SHALL** 사용자의 카테고리 accordion 펼침/접힘 상태를 `sessionStorage[watchtower_accordion_v1]`에 JSON으로 유지한다.

**탭 닫힘 후**: sessionStorage 자동 소실 → 새 탭/새 창에서는 default rule(구독 카테고리 expand / 기타 collapse) 적용.

**스토리지 제외 정책**: 구독·채널 등 알림 관련 상태는 절대 sessionStorage/localStorage 사용 불가 (Step 4 정책 유지). UI layout state만 허용.
